# Dalbit API Documentation

This document outlines the REST APIs provided by the Dalbit Spring Server.

## Authentication (Auth)

### 1. Social Login
- **Endpoint:** `POST /auth/login`
- **Description:** Authenticate users via social providers (e.g., KAKAO, GOOGLE).
- **Request Body:**
  - `provider` (String): Social login provider (e.g., "KAKAO").
  - `token` (String): Token received from the social authentication server.
- **Success Response (200 OK):**
  - `code` (String): Response status code.
  - `message` (String, Optional): Response message.
  - `data.accessToken` (String): Issued JWT access token.
  - `data.refreshToken` (String): Issued JWT refresh token.
- **Error Responses:**
  - `400 Bad Request`: Missing required input (e.g., provider or token).
  - `401 Unauthorized`: Invalid provider token.

### 2. Token Reissue
- **Endpoint:** `POST /auth/refresh`
- **Description:** Issue a new access token and refresh token using a valid refresh token.
- **Request Body:**
  - `refreshToken` (String): The previously issued refresh token.
- **Success Response (200 OK):**
  - `code` (String): Response status code.
  - `message` (String, Optional): Response message.
  - `data.accessToken` (String): Newly issued JWT access token.
  - `data.refreshToken` (String): Newly issued JWT refresh token.
- **Error Responses:**
  - `401 Unauthorized`: Invalid refresh token.

## User

### 1. Get User Information
- **Endpoint:** `GET /api/v1/dalbit/user/info`
- **Description:** Retrieve information about the authenticated user.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
  - `data.externalId` (String): User's unique identifier.
  - `data.name` (String): User's name.
  - `data.email` (String): User's email address.
- **Error Responses:**
  - `400 Bad Request`: User not found.

### 2. Update User Name
- **Endpoint:** `PATCH /api/v1/dalbit/user/update/name`
- **Description:** Update the name of the authenticated user.
- **Request Body:**
  - `name` (String): The new name for the user.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
- **Error Responses:**
  - `400 Bad Request`: Invalid input (e.g., empty name).

## AudioBook

### 1. Get My AudioBook List
- **Endpoint:** `GET /api/v1/dalbit/audio/info`
- **Description:** Retrieve the list of audiobooks generated by the user.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
  - `data[].audioBookExternalId` (String): Audiobook's unique identifier.
  - `data[].voiceExternalId` (String): Identifier of the voice used.
  - `data[].fairytaleId` (Number): Original fairytale's ID.
  - `data[].status` (String): Generation status (e.g., IN_PROGRESS, COMPLETED).
  - `data[].audioUrl` (String): URL to the generated audiobook file (e.g., S3 URL).

### 2. Delete AudioBook
- **Endpoint:** `DELETE /api/v1/dalbit/audio/delete`
- **Description:** Delete an audiobook by its external ID.
- **Request Body:**
  - `externalId` (String): Unique identifier of the audiobook to delete.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
- **Error Responses:**
  - `400 Bad Request`: Missing audiobook identifier.

### 3. Start AudioBook Generation
- **Endpoint:** `POST /api/v1/dalbit/audio/generate/start`
- **Description:** Request the creation of an audiobook from a specific fairytale using a trained voice.
- **Request Body:**
  - `fairytaleId` (Number): ID of the fairytale to generate an audiobook for.
  - `voiceExternalId` (String): Identifier of the trained voice to use.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
- **Error Responses:**
  - `400 Bad Request`: Voice does not exist or Fairytale does not exist.

## Voice

### 1. Get Voice List
- **Endpoint:** `GET /api/v1/dalbit/voice/info`
- **Description:** Retrieve the list of voices owned by the user.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
  - `data[].externalId` (String): Voice unique identifier.
  - `data[].name` (String): Voice name.

### 2. Get Voice Upload URLs
- **Endpoint:** `GET /api/v1/dalbit/voice/upload-url`
- **Description:** Get pre-signed URLs to upload voice data files.
- **Query Parameters:**
  - `externalId` (String): Target ID for the voice data to be uploaded.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
  - `data.uploadUrls[]` (List of Strings): List of S3 Presigned URLs for file uploads.

### 3. Register Voice
- **Endpoint:** `POST /api/v1/dalbit/voice/register`
- **Description:** Register a new voice profile.
- **Request Body:**
  - `name` (String): Name for the new voice.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
- **Error Responses:**
  - `409 Conflict`: Voice name already exists.

### 4. Update Voice Name
- **Endpoint:** `PATCH /api/v1/dalbit/voice/update/name`
- **Description:** Change the name of an existing voice.
- **Request Body:**
  - `externalId` (String): Unique identifier of the voice to update.
  - `name` (String): The new name for the voice.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
- **Error Responses:**
  - `400 Bad Request`: Voice does not exist.

### 5. Delete Voice
- **Endpoint:** `DELETE /api/v1/dalbit/voice/delete`
- **Description:** Delete a voice.
- **Request Body:**
  - `externalId` (String): Unique identifier of the voice to delete.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.

### 6. Start Voice Training
- **Endpoint:** `POST /api/v1/dalbit/voice/train/start`
- **Description:** Initiate the training process for a voice model.
- **Request Body:**
  - `externalId` (String): Unique identifier of the voice model to train.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
- **Error Responses:**
  - `400 Bad Request`: Voice does not exist or voice files (10 files) are not fully uploaded.

## Device

### 1. Get My Devices
- **Endpoint:** `GET /api/v1/dalbit/device/myDevices`
- **Description:** Retrieve the list of devices registered to the user.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
  - `data[].serialNumber` (String): Device serial number.
  - `data[].deviceType` (String): Type of the device.
  - `data[].name` (String): Device name.

### 2. Register Device Owner
- **Endpoint:** `POST /api/v1/dalbit/device/register/owner`
- **Description:** Register a user as the owner of a device.
- **Request Body:**
  - `serialNumber` (String): Serial number of the device to register.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
- **Error Responses:**
  - `409 Conflict`: Device is already registered to an owner.

### 3. Update Device Name
- **Endpoint:** `PATCH /api/v1/dalbit/device/update/name`
- **Description:** Update the name of a registered device.
- **Request Body:**
  - `serialNumber` (String): Device serial number.
  - `newName` (String): The new name for the device.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
- **Error Responses:**
  - `403 Forbidden`: No permission to modify the device (not the owner).

## User Device (FCM & Mobile Info)

### 1. Register User Device
- **Endpoint:** `POST /api/v1/dalbit/userDevice/register`
- **Description:** Register a user's mobile device (e.g., to receive push notifications).
- **Request Body:**
  - `deviceUniqueId` (String): Unique identifier for the device (e.g., UUID).
  - `fcmToken` (String): Issued FCM push token.
  - `osType` (String): Operating system type (e.g., "AOS", "IOS").
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
- **Error Responses:**
  - `400 Bad Request`: Missing required input (e.g., FCM Token).

### 2. Update FCM Token
- **Endpoint:** `PATCH /api/v1/dalbit/userDevice/update/token`
- **Description:** Update the FCM push token for a registered device.
- **Request Body:**
  - `deviceUniqueId` (String): Unique identifier for the device.
  - `fcmToken` (String): The new FCM push token.
- **Success Response (200 OK):**
  - `code` (String): Response code.
  - `message` (String): Response message.
- **Error Responses:**
  - `400 Bad Request`: User device does not exist.

## Internal / Queue Authentication (RabbitMQ)

### 1. Queue User Authentication
- **Endpoint:** `POST /internal/queue/auth/user`
- **Content-Type:** `application/x-www-form-urlencoded`
- **Description:** Authenticate a RabbitMQ connection.
- **Form Parameters:**
  - `userName` (String): RabbitMQ connection username (e.g., `usr_{externalId}` or `dev_{serialNumber}`).
  - `password` (String): Password (JWT token for user, Secret Key for device).
- **Responses (200 OK):**
  - Returns `allow` if authenticated.
  - Returns `deny` if authentication fails.

### 2. Queue Vhost Authorization
- **Endpoint:** `POST /internal/queue/auth/vhost`
- **Content-Type:** `application/x-www-form-urlencoded`
- **Description:** Authorize access to a Virtual Host. Always allowed in the current implementation.
- **Form Parameters:**
  - `vhost` (String): Target Virtual Host name.
  - `userName` (String, Optional): Connection username.
  - `ip` (String, Optional): Connection IP.
- **Response (200 OK):**
  - Returns `allow`.

### 3. Queue Resource (Topic) Authorization
- **Endpoint:** `POST /internal/queue/auth/resource`
- **Content-Type:** `application/x-www-form-urlencoded`
- **Description:** Authorize access to a specific RabbitMQ resource (e.g., read/write on a topic).
- **Form Parameters:**
  - `userName` (String): Connection username.
  - `resource` (String): Resource type (e.g., "topic", "queue").
  - `topicName` (String): Target topic name.
  - `permission` (String): Requested permission (e.g., "read", "write", "configure").
- **Responses (200 OK):**
  - Returns `allow` if authorized.
  - Returns `deny` if authorization fails.
